# Regras técnicas do projeto

## Arquitetura

- usar arquitetura modular
- separar domínio, aplicação e infraestrutura quando fizer sentido
- evitar lógica de negócio em controllers
- manter serviços pequenos e claros
- nomear arquivos e funções com clareza

## Backend

- usar NestJS
- usar DTOs para validação
- usar modules por domínio
- manter entidades consistentes com as regras de produto
- documentar decisões importantes no código quando necessário

## Frontend

- seguir o design e a estrutura gerados no v0
- preservar a identidade visual calma e minimalista
- evitar mudanças arbitrárias na UI sem necessidade funcional
- componentes devem ser reutilizáveis

## Produto

- nunca introduzir elementos de gamificação agressiva
- nunca destacar métricas de vaidade
- não criar feed infinito
- não transformar a plataforma em uma rede social de performance

## Qualidade

- código limpo e legível
- evitar overengineering
- priorizar consistência
- deixar o projeto pronto para expansão futura
